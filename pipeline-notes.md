# Pipeline Notes

## Five Stage Pipeline

1. IF - Instruction Fetch  
2. ID - Instruction Decode / Register Read  
3. EX - Execute  
4. MEM - Memory Access  
5. WB - Write Back

## Common Hazards

### Data Hazards
Occurs when one instruction depends on previous instruction output.

### Control Hazards
Occurs with branch instructions.

### Structural Hazards
Occurs when hardware resources are limited.

## Solutions

- Forwarding
- Stall cycles
- Branch prediction
- Better hardware design

## Key Learning

Pipelining improves throughput by overlapping instruction execution.
